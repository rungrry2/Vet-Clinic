🔥 วิธี “เรียกใช้ Trigger” (ของคุณ)
🧾 1. ทดสอบ tr_POS_Transaction_Stock_Update

👉 แค่ INSERT ลง POS_Transaction

INSERT INTO POS_Transaction (transaction_id, medicine_id, quantity)
VALUES ('T001', 'MED001', 2);
🎯 สิ่งที่จะเกิดขึ้นอัตโนมัติ:
✅ stock ลด
✅ ถ้า stock ไม่พอ → ERROR + rollback
✅ log เข้า Audit_Log
📦 2. ทดสอบ tr_Medicine_Stock_Audit
UPDATE Medicine_Stock
SET quantity = quantity + 10
WHERE medicine_id = 'MED001';
🎯 สิ่งที่จะเกิด:
✅ บันทึก log ก่อน/หลังใน Audit_Log
🐾 3. ทดสอบ tr_Pets_Validate
INSERT INTO Pets (pet_id, owner_id, species_id, pet_name, weight)
VALUES ('P999', 'O001', 'SP001', N'ตัวทดลอง', -5);
🎯 ผล:
ERROR: Pet weight cannot be negative

👉 ❌ insert ไม่เข้า

👤 4. ทดสอบ tr_Owners_Validate
INSERT INTO Owners (owner_id, first_name, last_name, email)
VALUES ('O999', 'Test', 'User', 'invalid-email');

👉 ❌ error ทันที

📅 5. ทดสอบ tr_Appointments_Validate

👉 ลอง insert ซ้ำเวลาเดียวกัน:

INSERT INTO Appointments (appointment_id, pet_id, vet_id, status_id, appointment_date, appointment_time)
VALUES ('A999', 'P001', 'V001', 'S001', '2026-03-26', '10:30');

👉 ถ้ามี slot นี้แล้ว:
❌ จะโดน block

🔍 ดูผลลัพธ์ (สำคัญมาก)
📜 ดู log
SELECT * FROM Audit_Log ORDER BY change_date DESC;
💡 สรุป logic ของ Trigger คุณ
Trigger	ทำอะไร
POS_Transaction	ตัด stock + log
Medicine_Stock	audit การเปลี่ยน
Pets	กัน weight ผิด
Owners	ตรวจ email
Appointments	กันเวลาซ้ำ
